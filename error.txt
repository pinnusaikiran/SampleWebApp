EB environments can only accept updates when the environment status is Ready.

Status can be Updating, Launching, Terminating, or Rolling Back.

If you just made configuration changes (like adding the EC2 instance profile), EB will be updating in the background.

CodePipeline sees this and cannot deploy until the environment is back to Ready.

Or Apply Configuration Changes if needed

Always ensure status = Ready before triggering the next pipeline deployment.

Right after deployment or restart, EB instances need 30–60 seconds to:

Start your Python app (Flask)

Start the health monitoring agent

Begin sending metrics to Elastic Beanstalk

During this short window, the health status may show “No Data” or “Info”.

Why It Happens

Single-instance environments start faster, but it still takes some seconds for EB to report.

If the app fails to start (e.g., WSGIPath misconfigured), EB will eventually report Red / Severe.

In your case, deployment logs show the app was deployed successfully.
